## Usage Notes

- All color classes support the standard Tailwind utilities: `bg-`, `text-`, `border-`, `outline-`, `ring-`, `divide-`, `placeholder-`, etc.
- Colors automatically adapt to the active brand (Brand A or Brand B) when the `.brand-a` or `.brand-b` class is applied to a parent element.
- Primitive colors (orange, green, grey, red, black, white) are brand-specific and may not work correctly when switching brands. Use semantic colors (primary, secondary, tertiary, success, error, warning, info, neutral) for brand-agnostic styling.
- Font family and weight variables are currently mapped to Brand A values. For full Brand B support, these would need to be updated in the configuration.

## Examples

```html
<!-- Semantic colors (work with both brands) -->
<div class="bg-primary-default text-neutral-white">Primary button</div>
<div class="bg-success-light text-success-dark">Success message</div>
<div class="border-error-default">Error border</div>

<!-- Typography -->
<h1 class="font-heading font-bold">Heading</h1>
<p class="font-body font-regular">Body text</p>
```
